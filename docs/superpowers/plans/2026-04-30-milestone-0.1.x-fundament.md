## Task 2: Packwiz einrichten

Packwiz ist das Standard-CLI-Tool für NeoForge-Modpacks auf Modrinth. Es generiert automatisch das mrpack-Manifest und verwaltet Mod-Versionen.

**Dateien:** `modrinth/pack.toml`, `modrinth/index.toml`

- [ ] **Schritt 1: Packwiz installieren**

Binary von https://github.com/packwiz/packwiz/releases herunterladen (Linux: `packwiz_linux_amd64`).

```bash
# Binary nach PATH verschieben
chmod +x packwiz_linux_amd64
sudo mv packwiz_linux_amd64 /usr/local/bin/packwiz

# Verifikation
packwiz version
# Erwartet: packwiz 1.x.x
```

- [ ] **Schritt 2: Pack initialisieren**

```bash
cd /home/vugas/minecraft-modpack/modrinth

packwiz init \
  --name "APEX Modpack" \
  --author "VugasDev" \
  --version "0.1.0-alpha" \
  --game-version "1.21.1" \
  --modloader neoforge \
  --modloader-version "21.1.0"
```

Erwartet: `pack.toml` und `index.toml` werden erstellt.

- [ ] **Schritt 3: Erstellte pack.toml prüfen**

`modrinth/pack.toml` sollte so aussehen:

```toml
name = "APEX Modpack"
author = "VugasDev"
version = "0.1.0-alpha"
pack-format = "packwiz:1.1.0"

[versions]
minecraft = "1.21.1"
neoforge = "21.1.0"
```

NeoForge-Version auf aktuelle Stable anpassen: https://projects.neoforged.net/neoforged/neoforge

## Task 4: World Gen konfigurieren

**Dateien:** `config/tectonic-common.toml`, `config/chunky/` (neu)

Tectonic aktiviert die 800-Höhe via Config. Chunky braucht ein Preset für den Pre-Gen-Radius.

- [ ] **Schritt 1: Tectonic Config erstellen**

Datei `config/tectonic-common.toml` erstellen:

```toml
[world]
# Aktiviert erweiterte Welthöhe (Standard Tectonic: 384, Extended: 800)
# Erfordert neue Welt — kann nicht auf bestehender Welt aktiviert werden
tallWorldHeight = true

[terrain]
# Tectonic Erosion für natürlichere Berge
erosion = true
```

> **Wichtig:** Im Spiel prüfen ob Tectonic die Config an diesem Pfad liest.
> Falls nicht: Tectonic Datapack nutzen (im World-Ordner unter `datapacks/`).

- [ ] **Schritt 2: Chunky Preset erstellen**

Verzeichnis und Preset erstellen:

```bash
mkdir -p /home/vugas/minecraft-modpack/config/chunky
```

Datei `config/chunky/default-preset.json`:

```json
{
  "version": 1,
  "tasks": [
    {
      "world": "minecraft:overworld",
      "shape": "square",
      "centerX": 0,
      "centerZ": 0,
      "radius": 3000
    },
    {
      "world": "minecraft:the_nether",
      "shape": "square",
      "centerX": 0,
      "centerZ": 0,
      "radius": 1500
    }
  ]
}
```

Radius 3000 = 6km × 6km Overworld (ausreichend für Early-bis-Midgame ohne spontane Lags).

## Task 5: NeoForge Server aufsetzen

**Datei:** `docs/server-setup.md` (neue Anleitung für Reproduzierbarkeit)

- [ ] **Schritt 1: Server-Setup-Doku erstellen**

Datei `docs/server-setup.md`:

```markdown
# Server Setup — APEX Modpack 0.1.x

## Voraussetzungen
- Java 21 (empfohlen: Eclipse Temurin 21)
- Mindestens 8 GB RAM für den Server-Prozess
- Ubuntu 22.04+ oder Windows Server

## NeoForge Server installieren

1. NeoForge 1.21.1 Installer herunterladen:
   https://projects.neoforged.net/neoforged/neoforge → Version für 1.21.1

2. Server installieren:
   ```bash
   java -jar neoforge-21.1.x-installer.jar --install-server ./server
   cd server
   ```

3. Startskript `start.sh` erstellen:
   ```bash
   #!/bin/bash
   java -Xmx8G -Xms4G \
     -XX:+UseG1GC \
     -XX:+ParallelRefProcEnabled \
     -XX:MaxGCPauseMillis=200 \
     -jar libraries/net/neoforged/neoforge/21.1.x/neoforge-21.1.x-server.jar nogui
   ```

4. EULA akzeptieren:
   ```bash
   echo "eula=true" > eula.txt
   ```

## Modpack deployen

```bash
# Mods aus mrpack extrahieren oder einzeln aus Modrinth laden
# config/ und kubejs/ aus dem Repo kopieren:
cp -r /path/to/repo/config ./config
cp -r /path/to/repo/kubejs ./kubejs
cp -r /path/to/repo/config/ftbquests ./config/ftbquests
```

## server.properties

```properties
max-players=10
view-distance=12
simulation-distance=8
online-mode=true
difficulty=normal
spawn-protection=0
```

## Erster Start & Chunky Pre-Gen

1. Server starten: `./start.sh`
2. Warten bis: `[Server thread/INFO]: Done (Xs)!`
3. Chunky Pre-Gen starten (in der Server-Konsole):
   ```
   chunky world minecraft:overworld
   chunky center 0 0
   chunky radius 3000
   chunky shape square
   chunky start
   ```
4. Status prüfen: `chunky progress`
5. Warten bis Pre-Gen abgeschlossen (kann 30–60 Minuten dauern)

## Verbindungstest

- Alle 5 Spieler verbinden sich gleichzeitig
- TPS überwachen: `/tps` sollte ≥18 zeigen
- Keine Timeout-Fehler in `logs/latest.log`
```
